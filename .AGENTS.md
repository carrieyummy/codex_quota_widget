# Project Rules

## 编码格式

文本类的文件均使用UTF-8作为默认编码格式

## 函数注释同步维护规则

在编写或修改任意主流编程语言代码时，函数、方法、过程、构造器、接口方法等可调用单元的注释维护要求如下：

1. 新增可调用单元时，必须添加相应注释或文档说明，至少包含：作用、参数说明、返回值/输出说明（如有）。
2. 参数说明应覆盖每个参数的含义、类型/格式要求（如有）、取值约束（如有）。
3. 修改已有代码逻辑时，必须同步检查并更新相关注释，避免注释与实现不一致。
4. 如为临时实现或限制条件，需在注释中明确标注，便于后续维护。
5. 注释语言默认中文，有特别说明用英文的除外。

## Python脚本Path规则

在编写 Python 代码时，路径处理遵循以下规则：

1. 优先使用 `pathlib.Path` 进行路径拼接、解析、遍历、存在性判断和文件读写。
2. 避免使用 `os.path`（如 `join`、`exists`、`dirname`、`basename`、`abspath`、`relpath` 等）。
3. 对必须接收字符串路径的旧接口，仅在边界处使用 `str(path_obj)` 进行转换。
4. 新增代码中不要引入 `import os.path`，也不要新增 `os.path.xxx` 的路径处理逻辑。

## `.code-workspace` 按需维护规则

VS Code 工作区配置相关要求如下：

1. 需求中若明确指定了某个 `.code-workspace` 文件，应在完成代码修改后检查该文件是否包含与本次修改相关的命令行启动配置、任务配置或调试参数。
2. 如果本次代码修改新增、删除、重命名或改变了命令行参数含义、默认值、必填性或取值约束，必须同步更新用户指定的 `.code-workspace` 文件中对应的参数配置。
3. 若 `.code-workspace` 文件使用 JSONC 注释说明参数用途，应同步维护注释，避免注释与当前命令行行为不一致。
4. 若代码修改未影响命令行参数，应保持 `.code-workspace` 文件不变，并在最终回复中说明无需更新的原因。
5. 最终回复中应说明是否更新了 `.code-workspace` 文件；如已更新，应概述同步的参数变化。
6. 注释默认用中文，有特别说明的除外。
